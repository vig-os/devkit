# [PR 510](https://github.com/vig-os/devcontainer/pull/510) fix: resolve draft releases in promote-release validation

## Description

The promote-release validate step queried `GET /repos/{owner}/{repo}/releases/tags/{tag}`, which does not return draft releases (404). The workflow now uses the paginated releases list API and jq to select the release by `tag_name`, so draft releases created by `release.yml` are found. The RC git tag cleanup step uses the same lookup so “release exists” stays correct for drafts if they ever apply.

`gh api` doesn’t support passing `jq` variables via `--arg`, so `$tag` in the `--jq` filter will be undefined and this call is expected to fail (often with a jq compile error). Consider removing `--jq/--arg` from `gh api` and instead piping the paginated JSON through `jq -s` (or similar) and using `--arg tag "$VERSION"` on the `jq` invocation.

# [PR 614](https://github.com/vig-os/devcontainer/pull/614) fix(smoke-test): idempotent release pipeline across candidate→final

## Description

Makes the smoke-test release pipeline idempotent when one base version `X.Y.Z` is
exercised by both a candidate pass (`X.Y.Z-rcN`) and a final pass (`X.Y.Z`).
Previously the `## [X.Y.Z]` CHANGELOG heading could end up dated before the
downstream `release-core` validation (which requires the `## [X.Y.Z] - TBD`
placeholder), failing with `does not contain '## [X.Y.Z] - TBD'` and requiring
manual CHANGELOG recovery (as happened for 0.3.7).

Defense-in-depth across the Python SSoT (`vig-utils`) and the synced workflow
templates.

## Type of Change

- [x] `fix` -- Bug fix

## Changes Made

**`prepare_changelog.py` (vig-utils, the tested SSoT):**
- `finalize` is now a **no-op** when the version heading is already dated (the
  linked `## [X.Y.Z](…) - YYYY-MM-DD` form this tool writes), instead of erroring.
  A plain historical dated heading is still rejected.
- `prepare` **dedupes** an existing same-version heading (TBD or dated): it folds
  the existing section's bullets back into the freshly-frozen content and drops it,
  yielding exactly one `## [X.Y.Z] - TBD` heading instead of stacking a duplicate.
- New **`reset-version <version>`** command reverts a dated heading back to `- TBD`
  (idempotent; no-op when already TBD or absent).

**Synced workflow templates:**
- `repository-dispatch.yml`: the post-seed verification is now **scoped to the
  `## Unreleased` section** (awk), so a bullet stranded in a stale `[X.Y.Z]`
  section can no longer mask an un-seeded Unreleased.
- `prepare-release.yml`: runs `prepare-changelog reset-version` immediately before
  `prepare`, normalizing a dated heading to `- TBD` at freeze time (in-container,
  where the CLI is on PATH).

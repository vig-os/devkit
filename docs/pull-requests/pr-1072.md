# [PR 1072](https://github.com/vig-os/devkit/pull/1072) fix(security): except gawk 5.4.0 CVE batch pending nixpkgs 5.4.1 bump

## Summary

Unblocks the **1.2.0-rc1** publish, which failed at the blocking Vulnix CVE Gate ([run 29346662524](https://github.com/vig-os/devkit/actions/runs/29346662524)) on four gawk 5.4.0 findings disclosed by CERT-PL on 2026-07-13 ([advisory](https://cert.pl/en/posts/2026/07/CVE-2026-40467/)):

| CVE | CVSS | What | Reachability here |
|-----|------|------|-------------------|
| CVE-2026-40468 | 9.1 | integer overflow, `builtin.c` | developer/CI-chosen awk input only |
| CVE-2026-40469 | 9.1 | integer overflow, `do_sub()` | **32-bit builds only** — image is 64-bit |
| CVE-2026-40467 | 7.5 | UAF, `do_getline_redir()` | crash only, dev-chosen input |
| CVE-2026-40553 | 7.5 | stack overflow, `readdir` ext | needs opt-in `-l readdir`, unused here |

## Why an exception and not a bump

All four are fixed upstream in **gawk 5.4.1**. The nixpkgs bump ([NixOS/nixpkgs#540158](https://github.com/NixOS/nixpkgs/pull/540158)) merged to `staging` on 2026-07-12 (gawk is a stdenv mass-rebuild) and has not reached `nixos-26.05` — nor `release-26.05`, `staging-26.05`, `master`, or `nixpkgs-unstable` (verified 2026-07-14). The "advance the rev" lever has nowhere to land, so this follows the exception-register model (precedents #941, #963): short-dated block (**expires 2026-07-28**), per-CVE rationale, dropped in favour of a pin advance once 5.4.1 lands.

## Verification

- Full `just precommit` suite green (incl. `check-expirations`, `sync-manifest`).
- `vulnix-gate` run locally against the updated register with a findings file reproducing the four CVEs: **exit 0**; negative control with an unregistered CVE still fails (exit 1).

# [PR 1006](https://github.com/vig-os/devkit/pull/1006) refactor(ci): make the release/automation workflow set mode-aware

## Summary

PART B of #991: converts the eight container-coupled scaffold workflows to the
Option-A pattern (ADR: docs/rfcs/ADR-conditional-container-toolchain.md).
Release choreography unchanged — toolchain provisioning only.

| Workflow | Conversion |
|---|---|
| release.yml | resolve-image job → resolve-toolchain; threads toolchain_mode/toolchain_image/devkit_version into the reusable calls; rollback job on conditional container + composite |
| release-core.yml | own resolve-image job deleted; validate/finalize/test on `container: image: ${{ inputs.toolchain_image }}` + composite; image_tag output now sourced from devkit_version |
| release-publish.yml | same (inputs-driven) |
| release-extension.yml | untouched (host-native, consumer-owned) |
| prepare-release.yml | validate runs resolve-toolchain inline (host step), prepare consumes its outputs |
| promote-release.yml | resolve-toolchain job; 4 jobs converted |
| sync-main-to-dev.yml | resolve-toolchain job; 2 jobs converted |
| renovate-changelog-build.yml | converted; renovate-bot gates kept verbatim |
| sync-issues.yml | converted |

Also: scaffold `resolve-image` action retired (manifest sync entry dropped —
the devkit's own root copy stays for #996); job-level `UV_PROJECT_ENVIRONMENT`
env and inline safe.directory / prek-guard steps removed (composite owns them);
MIGRATION.md supported-boundary rewritten; DOWNSTREAM_RELEASE.md provisioning
note; CHANGELOG entry.

# [PR 592](https://github.com/vig-os/devcontainer/pull/592) fix(install): point install/upgrade at canonical raw URL piped to bash

## Summary

Fixes #591. The install/upgrade flow pointed users at `https://vig-os.github.io/devcontainer/install.sh`, which **404s** because GitHub Pages was never enabled for this repo, and piped it to `sh` despite `install.sh` having a `#!/bin/bash` shebang and using bashisms.

All references now use the canonical form already in `README.md` (single source of truth):

```
curl -sSf https://raw.githubusercontent.com/vig-os/devcontainer/main/install.sh | bash -s -- --force ./
```

Two defects addressed:
1. **Wrong host** тАФ `vig-os.github.io` (Pages, never enabled) тЖТ `raw.githubusercontent.com/vig-os/devcontainer/main`.
2. **Wrong shell** тАФ `| sh` тЖТ `| bash`.

## Changes

- `assets/workspace/.devcontainer/justfile.devc` тАФ the real `just devcontainer-upgrade` host command + the in-container error hint
- `assets/workspace/.devcontainer/scripts/version-check.sh` тАФ upgrade-available nag message
- `assets/smoke-test/README.md` тАФ smoke-test install instructions
- `install.sh` тАФ usage/help comments and `--help` output (host on the `-o install.sh` download line, `sh`тЖТ`bash` on all installer pipes)
- `CHANGELOG.md` тАФ Unreleased / Fixed entry (synced to the workspace asset by the manifest hook)

The unrelated `get.docker.com | sh` and `astral.sh/uv/install.sh | sh` installer pipes are left untouched.

## Verification

- New URL returns **HTTP 200**: `curl -sSf https://raw.githubusercontent.com/vig-os/devcontainer/main/install.sh`
- `bash -n` syntax check passes on `install.sh` and `version-check.sh`
- `pre-commit run --all-files` (shellcheck, pymarkdown, sync-manifest, agent-identity) passes
- `pytest`: 714 passed, 29 skipped. The single `test_manifest_files` failure was a stale **local** container image baking the pre-edit CHANGELOG; source and synced asset hash identically, so CI's image rebuild resolves it.

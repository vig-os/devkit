# [PR 447](https://github.com/vig-os/devcontainer/pull/447) chore(ci): remove PR Title Check and validate-commit subject-only mode

## Description

Removes the **PR Title Check** GitHub Actions workflow that produced false positives (e.g. on release PRs) and drops the **`--subject-only`** mode from `validate-commit-msg`, which existed only for that CI path. Full commit messages remain validated by pre-commit and `validate-commit-msg` as before.

`test_chore_minimal_with_blank_line_only` / its docstring claim this is validating “subject + blank line only”, but the validator currently `rstrip()`s the content, so the trailing blank line is removed before `splitlines()`. As written, this test passes because chores are accepted as subject-only, not because a blank separator is preserved. Consider renaming the test to reflect subject-only acceptance, or updating the validator to preserve/enforce the blank-line separator if that’s the intended rule.
```suggestion
    def test_chore_minimal_subject_only(self):
        """Minimal chore commit: subject line only (trailing blank lines are ignored)."""
```

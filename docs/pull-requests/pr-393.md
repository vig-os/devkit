# [PR 393](https://github.com/vig-os/devcontainer/pull/393) fix: add git editor fallback in devcontainer setup

## Description

Fixes the devcontainer bug where `git commit` without `-m` can fail if a host-provided `core.editor` is unavailable inside the container. The setup script now applies a safe fallback editor only when needed, and regression coverage validates the behavior.

This test overwrites `.devcontainer/.conf/.gitconfig` and re-runs `setup-git-conf.sh`, which will overwrite the container user’s `~/.gitconfig` for the remainder of the session (the `devcontainer_up` fixture is session-scoped). That can change state for later tests (e.g., git signing config) and make the suite order-dependent. Consider running the script with an isolated `HOME` (so `git config --global` writes to a temp gitconfig), or back up and restore both the `.conf/.gitconfig` and `~/.gitconfig` contents at the end of this test.
```suggestion
                "orig_conf=.devcontainer/.conf/.gitconfig && "
                "bak_conf=.devcontainer/.conf/.gitconfig.test-bak && "
                "[ -f \"$orig_conf\" ] && cp \"$orig_conf\" \"$bak_conf\" || true && "
                "printf '[core]\\n\\teditor = does-not-exist-editor\\n' > \"$orig_conf\" && "
                "export HOME=/tmp/setup-git-conf-home && "
                "rm -rf \"$HOME\" && mkdir -p \"$HOME\" && "
                ".devcontainer/scripts/setup-git-conf.sh >/tmp/setup-git-conf.log 2>&1 && "
                "git config --global --get core.editor && "
                "rm -rf \"$HOME\" && "
                "[ -f \"$bak_conf\" ] && mv \"$bak_conf\" \"$orig_conf\" || rm -f \"$orig_conf\""
```

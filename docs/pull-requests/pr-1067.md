# [PR 1067](https://github.com/vig-os/devkit/pull/1067) feat(ci): add prepare-release extension hook with devkit dogfooding

## Summary

Implements #1059 — the mutating counterpart to `release-extension.yml`.

### The hook
- New scaffolded `prepare-release-extension.yml` (workflow_call, default no-op, **preserved** file — added to `PRESERVE_FILES`, carries the seeded banner): inputs `version`, `release_branch`, `branch_sha` (post-freeze), `dry_run`, git identity; `secrets: inherit` so real extensions can mint the COMMIT_APP token exactly like the freeze commit does.
- `prepare-release.yml` restructured into `validate → prepare → extension → open-pr` plus a single `rollback` job (mirroring `release.yml`'s pattern): the extension runs strictly **after** the `release/X.Y.Z` branch exists and **before** the draft PR opens, so extension commits appear in the PR diff from the start, and extension failure lands in the existing rollback domain (delete branch + restore dev changelog — no new rollback machinery).

### Devkit dogfooding — its one template divergence removed
Devkit's hardcoded `sync_manifest.py` step (and its rollback twin) moves into devkit's own `prepare-release-extension.yml`. Both `prepare-release.yml` copies now share a structurally identical, test-pinned job DAG; remaining diffs are the known pre-existing toolchain divergences (#991/#996).

### Drift window closed (found during implementation)
The removed inline step also kept dev's hook-synced changelog mirror consistent with the freeze commit. Devkit's extension therefore ends with a dev-side reconciliation: regenerate the mirror against dev's frozen changelog and commit **just the mirror** to dev (idempotent, dry-run-gated). The workflow header documents a full failure-ordering analysis: the reconcile commit is the last failable-adjacent step, the rollback's SHA guard composes correctly with it, and every enumerated failure path leaves dev with root+mirror consistent — without teaching the scaffold-shaped rollback any devkit-specific paths.

### Docs
Hook contract added to `docs/DOWNSTREAM_RELEASE.md` (dry-run + rollback semantics, and the commit-action dist-rebuild example — its first consumer, closing the manual rebuild loop from vig-os/commit-action#71).

TDD: two failing-test → implementation pairs; 13 shape/DAG tests including the devkit-vs-scaffold structural identity pins.

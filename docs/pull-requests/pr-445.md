The current check treats any `gh release view` failure (auth/rate-limit/network) as “release does not exist” because stderr is discarded and only exit code 0 is handled. That can let the workflow proceed even when the GitHub API call failed, defeating the purpose of an early validation guard. Consider using `retry` and inspecting the error (or using `gh api repos/$GITHUB_REPOSITORY/releases/tags/$PUBLISH_VERSION`) so that only a 404/not-found is treated as “does not exist”, and all other errors fail the job with a clear message.
```suggestion
          # Verify via GitHub API that no release exists for this tag.
          # Only a 404/not-found is treated as "does not exist"; all other errors fail the job.
          attempts=3
          delay_seconds=5
          
          for attempt in $(seq 1 "$attempts"); do
            echo "Checking for existing GitHub Release (attempt ${attempt}/${attempts})..."
            
            # Capture both output and exit status from gh api.
            # We do not redirect stderr away so that we can inspect the error.
            api_output="$(gh api "repos/${GITHUB_REPOSITORY}/releases/tags/${PUBLISH_VERSION}" 2>&1)"
            status=$?
            
            if [ "$status" -eq 0 ]; then
              echo "ERROR: GitHub Release already exists for $PUBLISH_VERSION"
              echo "Delete it manually before retrying: gh release delete $PUBLISH_VERSION --yes"
              exit 1
            fi
            
            # gh api returns non-zero for HTTP errors; detect a 404/not-found.
            if printf '%s\n' "$api_output" | grep -qi '404'; then
              echo "No existing GitHub Release found for $PUBLISH_VERSION (received HTTP 404)."
              exit 0
            fi
            
            echo "Warning: Failed to verify GitHub Release for $PUBLISH_VERSION (attempt ${attempt}/${attempts})." >&2
            echo "gh api output:" >&2
            printf '%s\n' "$api_output" >&2
            
            if [ "$attempt" -lt "$attempts" ]; then
              echo "Retrying in ${delay_seconds}s..." >&2
              sleep "$delay_seconds"
            else
              echo "ERROR: Unable to verify whether a GitHub Release exists for $PUBLISH_VERSION after ${attempts} attempts." >&2
              echo "Aborting to avoid proceeding with an unsafe or ambiguous state." >&2
              exit 1
            fi
          done
```

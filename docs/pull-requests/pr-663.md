# [PR 663](https://github.com/vig-os/devcontainer/pull/663) feat(scripts): install/init delivery-mode picker — devcontainer|direnv|both (#641)

## Summary

Implements **T4.3 (#641)**: users choose how the dev environment is delivered.

- **`install.sh`** + **`assets/init-workspace.sh`** gain a `--mode devcontainer|direnv|both` flag (both `--mode X` and `--mode=X`), validated up front. When unset: interactive prompt (default **both**); under `--no-prompts`/`--smoke-test`, defaults to **both** (preserves prior behaviour).
- **Prune after rsync** (scoped to the workspace, idempotent):
  - `devcontainer` → `.devcontainer/` only (removes `flake.nix` + `.envrc`)
  - `direnv` → `flake.nix` + `.envrc` only (removes `.devcontainer/`)
  - `both` → everything
- **Docs:** `README.md` + `CONTRIBUTE.md` (via `.j2` templates) document the flag, default, and prompt copy.

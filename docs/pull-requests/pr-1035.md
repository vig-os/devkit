# [PR 1035](https://github.com/vig-os/devkit/pull/1035) fix(setup): render language-aware scaffold .gitignore and codeql matrix

## Summary

Bundles the two "managed scaffold static is Python-hardcoded" bugs from the commit-action pilot — both are fixed by one shared language-detection mechanism, so they share a PR and a CI run.

### Shared mechanism
`init-workspace.sh` populates a `DETECTED_LANGUAGES` array from consumer marker files (`pyproject.toml` → python, `package.json` → node, `Cargo.toml` → rust) before the template copy, and renders the managed statics from it after the copy. Detection re-runs on every (re)scaffold, so the result is **upgrade-persistent** — the core failure both issues reported.

### #1024 — language-aware `.gitignore`
- `assets/workspace/.gitignore` is now a language-neutral base; per-language fragments live in `assets/gitignore.d/` (image-level, beside the installer — never shipped into the consumer tree) and are appended at install time.
- Node consumers get `node_modules/`, `*.tsbuildinfo`, `coverage/`, `.nyc_output/` and **no blanket `dist/`** (JS Actions commit `dist/index.js`); `dist/` moves into the Python fragment.
- Chosen over a preserved `.gitignore.project` companion because base+fragment re-assembly auto-tracks devkit ignore evolution across upgrades.

### #1025 — language-aware CodeQL matrix + default-setup conflict
- The installer rewrites the `language:` matrix line per detected language (python → `python`, node → `javascript-typescript`, rust → omitted pending CodeQL support; `actions` always analyzed, so the matrix is never empty). The committed template stays `['python', 'actions']` — correct for the devkit itself and SSoT-synced from the root workflow.
- The advanced-config vs GitHub default code-scanning setup conflict is documented in the workflow header (both copies) **and** printed as an install-time note. The installer never flips the repo's code-scanning API setting.

TDD: each fix lands as failing-test commit → implementation commit (7 new end-to-end bats tests).

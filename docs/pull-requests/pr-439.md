# [PR 439](https://github.com/vig-os/devcontainer/pull/439) fix(ci): accept bot PR approval when reviewDecision is empty

## Description

Release validation used only `reviewDecision` from `gh pr list`. When smoke-test orchestration approves the release PR with `git*ub-actions[bot]`, branch protection can leave `reviewDecision` empty while individual reviews still show `APPROVED`. This change falls back to the REST reviews API when `reviewDecision` is not `APPROVED`, matching the RCA on #438.

`gh api --paginate` combined with `--jq` will apply the jq filter per page, producing multiple outputs (one per page). That can make `APPROVED_COUNT` non-numeric (e.g., multiple lines) and break the `-eq` comparison under `set -e`. Consider using `--paginate --slurp` with a jq expression that flattens pages (or pipe to `jq -s` and sum) so the result is a single integer count across all pages.
```suggestion
              "repos/${GITHUB_REPOSITORY}/pulls/${PR_NUMBER}/reviews" | \
              jq -s '[.[][] | select(.state == "APPROVED")] | length')
```

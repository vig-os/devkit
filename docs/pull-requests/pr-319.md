The membership check uses `grep -Eq` with `$expected_doc` interpolated directly into a regex. Since filenames like `README.md` contain `.` (regex wildcard) and could contain other regex metacharacters, this can produce false matches and weaken the guardrail. Use a non-regex comparison (e.g., `case " $FILE_PATHS " in *" $expected_doc "* )` or split `CHANGED_FILES` and compare exact strings) or escape the filename before using it in a regex.

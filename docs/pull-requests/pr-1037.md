# [PR 1037](https://github.com/vig-os/devkit/pull/1037) fix(ci): drop ref dev from the sync-main-to-dev job checkout

## Summary

Fixes the bootstrap deadlock from #1034 (observed live in vig-os/commit-action adopting devkit 1.1.0): the `sync` job checked out `ref: dev` and then invoked a **local** composite action that may only exist on `main` — GitHub resolves `uses: ./...` against the checked-out workspace, so the job died the moment `main` introduced a local action `dev` didn't have yet, and the only thing that would carry it to `dev` was the sync PR the job could no longer open.

Fix per the issue's proposal: drop `ref: dev` from the checkout (workspace = the triggering `main` SHA, where the action is guaranteed to exist) in **both copies** — devkit's own workflow and the scaffold template. The two files are not manifest-linked (verified against `scripts/manifest.toml`), so both are edited directly.

Verified end-to-end that no subsequent step in either `sync` job depends on the local working tree: every step fetches and operates on `origin/main`/`origin/dev` refs (`rev-list`, `merge-tree`, `checkout -b ... origin/main`) or the gh API — the ref switch is behavior-preserving.

TDD: new YAML-shape test `tests/test_workflow_sync_checkout.py` (parametrized over both copies), RED before the fix, GREEN after.

Not included (follow-up candidate per the issue): a scaffold-lint rule "a job that checks out a non-default ref must not `uses: ./...`".

## Test evidence
- `uv run pytest tests/test_workflow_sync_checkout.py` → 2 passed (confirmed RED first)
- `bats tests/bats/just.bats` composite-usage assertion still holds
- `prek run --all-files` → green (re-verified independently before push)

Closes #1034

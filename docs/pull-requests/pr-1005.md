# [PR 1005](https://github.com/vig-os/devkit/pull/1005) refactor(ci): collapse per-mode ci.yml overlays into one mode-aware workflow

## Summary

Part A of #991: collapse the three per-mode `ci.yml` overlays (container /
`direnv` / `bare`) into **one mode-aware workflow**, built on the composites
merged in #1002 and the Option A verdict of
`docs/rfcs/ADR-conditional-container-toolchain.md`.

A leading `resolve-toolchain` job reads `.vig-os` and outputs the delivery
`mode`, container `image`, and `image-tag`. Every downstream job declares
`container: image: ${{ needs.resolve-toolchain.outputs.image }}` — the
devcontainer image in container-ish modes, an **empty string** in `direnv`/`bare`
that makes the job run on the host runner (proven by the #992 spike) — carries
the GHCR `credentials:` block unconditionally (inert on the host), and calls the
`setup-devkit-toolchain` composite as its first step. After that preamble, plain
`just sync|precommit|test` runs identically in every mode (no `nix develop -c`
prefixes).

The former per-mode overlay trees `assets/workspace-direnv/` and
`assets/workspace-bare/` and their `init-workspace.sh` deployment blocks are
removed. The rendered `ci.yml` is now byte-identical across all four modes.

## Design notes

- **Removed from the jobs** what the composite now owns: the job-level `env:`
  (`PREK_HOME` / `UV_PROJECT_ENVIRONMENT`), the "Fix git safe.directory" step,
  and the inline prek version-skew guard step.
- **Kept:** triggers, `permissions` (incl. `packages: read` for container mode),
  the `test-suite` dispatch gating `if:`s, the `image-tag` dispatch override
  (threaded into `resolve-toolchain`), and the summary job (restored to
  `needs: [resolve-toolchain, lint, test]` with a `resolve-toolchain.result`
  check like the container variant).
- **Timeouts:** the more generous of the variants where they differed —
  `resolve-toolchain` 2, `lint` 15 (was direnv's 15), `test` 20 (direnv's 20),
  `summary` 5.
- **Header comment** rewritten to describe the mode-aware design, referencing the
  ADR and #991.

## Tests

Reworked the #854/#885 per-mode `ci.yml` assertions into a single loop over all
four modes asserting the rendered `ci.yml`: wires `./.github/actions/resolve-toolchain`
and `./.github/actions/setup-devkit-toolchain`; selects the container via the
`needs.resolve-toolchain.outputs.image` expression; carries **no** hardcoded
`ghcr.io/vig-os/devcontainer` literal, no `resolve-image`, no inline `nix develop`
/ container-only env / prek guard; every external `uses:` SHA-pinned (local `./`
refs exempt, matching `check-action-pins`). Added an assertion that
`init-workspace.sh` no longer references the overlay dirs, and updated the bare
upgrade test. Dropped the now-obsolete overlay-specific tests. The #995
`actionlint` fixtures and #994 composite tests stay green over the new tree.

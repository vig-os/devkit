# [PR 401](https://github.com/vig-os/devcontainer/pull/401) fix(ci): require app tokens in downstream release templates

The docs list required app secrets, but they don't specify the required GitHub App permission set for the Commit/Release apps (e.g., Commit App needing contents write + branch protection bypass; Release App needing pull-requests/issues/releases/workflows as used by the templates). Without a documented permission matrix, downstream repos can provide the secrets but still fail at runtime with 403s. Consider documenting the minimum app permissions needed by each template workflow.
```suggestion

### Required GitHub App Permissions

The Commit and Release Apps must be installed on the downstream repository with at least the following repository permissions. Missing any of these can cause 403 errors even when the secrets are configured correctly.

**Commit App (used for branch/tag writes and protected refs)**

- Contents: **Read and write** (required for pushing commits and tags)
- Branch protection / Administration: permission level that allows **bypassing branch protection** for the release branch and tag updates, consistent with your org policy

**Release App (used for orchestration and GitHub API operations)**

- Contents: **Read and write** (for creating tags and releases)
- Pull requests: **Read and write** (for creating/updating release PRs if used by templates)
- Issues: **Read and write** (for creating failure/issues and sync-issues)
- Actions: **Read and write** (for interacting with workflow runs where applicable)
- Workflows: **Read and write** (for dispatching or updating workflows if required by local extensions)

If your organization enforces stricter policies, you may grant additional permissions as needed, but these are the minimum required for the shipped templates to function without authorization failures.
```

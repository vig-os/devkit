# [PR 431](https://github.com/vig-os/devcontainer/pull/431) fix: approve smoke-test release PR before downstream release

## Description

Smoke-test `repository_dispatch` orchestration marks the release PR ready and dispatches `release.yml`, but `release-core.yml` requires `reviewDecision == APPROVED`. The release app cannot self-approve its own PR. This PR adds a `ready-release-pr` job step that approves the release PR using `github.token`, with job-scoped `pull-requests: write`, so downstream Release Core validation passes without manual approval.

`gh pr review --approve` using `github.token` will fail in repos where Actions is not allowed to create/approve PR reviews (Actions settings: “Allow GitHub Actions to create and approve pull requests”). Consider adding a small guard that detects this specific 403/permission failure and prints a clear remediation message so smoke-test dispatch failures are self-explanatory.
```suggestion
          set -euo pipefail
          if ! OUTPUT="$(gh pr review "${PR_NUMBER}" --approve \
            --body "Automated approval by smoke-test dispatch orchestration." 2>&1)"; then
            printf '%s\n' "${OUTPUT}"

            if printf '%s' "${OUTPUT}" | grep -qiE 'resource not accessible by integration|http 403'; then
              echo "::error::Failed to auto-approve PR #${PR_NUMBER}. This is likely because the repository/organization setting 'Allow GitHub Actions to create and approve pull requests' is disabled. Enable this setting for this repository (or organization) or approve the PR manually."
            else
              echo "::error::Unexpected error while trying to auto-approve PR #${PR_NUMBER} via gh pr review. See output above for details."
            fi

            exit 1
          fi
```

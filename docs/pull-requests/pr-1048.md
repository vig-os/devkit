- **Second interpreter assumption found and fixed during testing:** `vig-utils` (a devTools entry, built against the devkit's pinned 3.14) propagates its interpreter onto PATH, shadowing an overridden `python3`. - Documented in `docs/MIGRATION.md` with the issue's FreeCAD snippet (`mkProjectShell { python = pkgs.python313; extraPackages = [ pkgs.freecad ]; }`).

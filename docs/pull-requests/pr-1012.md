# [PR 1012](https://github.com/vig-os/devkit/pull/1012) refactor(ci): migrate devkit-own workflows off the container/resolve-image pattern

## Summary

Migrates the three devkit-own workflows that still ran `container: ghcr.io/vig-os/devcontainer:<tag>` via a `resolve-image` job onto host runners provisioning the toolchain via `./.github/actions/setup-env` (host+Nix), matching ci/release/prepare-release/promote-release. Deletes the now-unused `resolve-image` action.

Closes the last consumers of the container/resolve-image pattern in this repo's own CI.

## Per-workflow conversion notes

### `sync-main-to-dev.yml`
- Deleted the `resolve-image` job.
- `check` and `sync` jobs now run on `ubuntu-24.04` host with a `Set up environment` (`setup-env`) step after checkout. `setup-env` provides git/gh/jq plus the `retry` bash function via its `BASH_ENV` shim, so the existing bare `retry ...` calls in `run:` blocks are unchanged.
- Dropped both `container:` blocks and both `Fix git safe.directory` steps (host runner owns `$GITHUB_WORKSPACE`, so git's `safe.directory` guard does not apply).
- Fixed `needs:` — `check` drops `needs: [resolve-image]`; `sync` goes `needs: [resolve-image, check]` -> `needs: [check]`. No `${{ needs.resolve-image... }}` expressions remain.

### `sync-issues.yml`
- Deleted the `resolve-image` job; `sync` drops `needs: [resolve-image]` and its `container:` block.
- Added `Set up environment` after checkout (provides `retry` for the cache-deletion step). Removed `Fix git safe.directory`. Job `permissions`/`concurrency`/`env` untouched.

### `renovate-changelog-build.yml` (root)
- Deleted the `resolve-image` job; single `build` job now runs on host with `setup-env` (`sync-dependencies: 'true'` so the project venv exists) after checkout.
- Dropped the `container:` + GHCR `credentials:` block and the `packages: read` permission (only needed for the image pull).
- `renovate-changelog-pr` -> `uv run renovate-changelog-pr` (the console script is not on `PATH` on the host; forced change). Root keeps its `assets/workspace/.devcontainer/CHANGELOG.md` mirror plumbing unchanged.

## Manifest decision (renovate-changelog-build.yml): option (b) — de-couple

`renovate-changelog-build.yml` was synced to the scaffold via `scripts/manifest.toml` ReplaceBlock/Sed transforms that rewrote the container-only root into the mode-aware `resolve-toolchain` + `setup-devkit-toolchain` template variant.

Chose **(b) de-couple**: the converted root has no `resolve-image`/`resolve-toolchain` job and no `container:` block, while the scaffold variant needs both plus the `setup-devkit-toolchain` composite. Deriving the template from the converted root would require re-synthesizing an entire job and the container block the root no longer has — i.e. re-authoring the majority of the file inside TOML. So the manifest entry + all its transforms are dropped and the scaffold copy at `assets/workspace/.github/workflows/renovate-changelog-build.yml` is maintained as an independently-authored file (precedent: #991 dropped the resolve-image action sync entry). The template file content is unchanged.

## resolve-image action removal

Deleted `.github/actions/resolve-image/` and its unit tests `tests/test_resolve_image_probe.py` / `tests/test_resolve_image_manifest.py` (they load the deleted action YAML). Verified `grep -rn resolve-image .github/` returns nothing. Updated the stale reference in `tests/bats/manifest-parsers.bats`.

## Verification

- `nix develop -c prek run --all-files` — green, exit 0 (actionlint validates all three restructured workflows; yamllint/shellcheck/sync-manifest all pass).
- `git diff origin/dev -- assets/workspace/` — **empty**. `uv run python scripts/sync_manifest.py sync assets/workspace/` produces no diff; the scaffold template is byte-identical to dev.
- `uv run pytest tests/test_transforms.py -q` — 6 passed (repointed the no-mirror-leak guard at the committed scaffold file under (b)).
- `nix develop -c bats tests/bats/manifest-parsers.bats tests/bats/just.bats` — green (54 + parser tests pass).
- `sync-main-to-dev` (push-to-main) and `sync-issues` (schedule) cannot be executed locally; actionlint + review is the local bar. This draft PR to `dev` exercises the repo's automated CI.

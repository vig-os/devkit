# [PR 662](https://github.com/vig-os/devcontainer/pull/662) feat(nix): downstream flake stub (non-overwriting) + nix2container pattern (#640)

## Summary

Implements **T4.2 (#640)**: downstream repos consume the shared toolchain as a flake **input** (so "update the dev env" = bump the input, never overwrites user files), plus the `nix2container` production-image pattern.

- **`assets/workspace/flake.nix`** — minimal stub: `vigos.url = github:vig-os/devcontainer`, `nixpkgs.follows = vigos/nixpkgs`, `flake-utils.follows = vigos/flake-utils`, `devShells.default = vigos.lib.mkProjectShell { … extraPackages }` with a placeholder `extraPackages` block (the user's, never overwritten).
- **`assets/workspace/.envrc`** — `use flake` via nix-direnv (mirrors the root onboarding from #633).
- **Never-overwrite** — added `flake.nix` + `.envrc` to `PRESERVE_FILES` in `init-workspace.sh` (same class as `justfile.project`). Reconciliation: the issue named `manifest.toml`/`sync_manifest.py`, but that path only mirrors **repo→template** and always overwrites — the actual scaffold-once guarantee for downstream user edits is `PRESERVE_FILES` (consulted by the `--force` upgrade rsync). Also un-ignored the template `.envrc` in `assets/workspace/.gitignore` (keeping `.direnv/` + `.envrc.local` ignored) and broadened the shellcheck exclude to `.envrc` at any path.
- **`nix2container`** — `docs/NIX2CONTAINER.md` + a buildable `examples/nix2container-production/` deriving a minimal runtime image from the same pinned `nixpkgs` (distinct from the devcontainer's `buildLayeredImage`). Future modular language shells noted as an opt-in extension point.

## Verified locally
- Downstream stub `devShells.default` **evaluates** against the epic flake (`--override-input vigos path:…` → `nix-shell`).
- nix2container example **evaluates** to a valid image derivation.
- New `init-workspace.bats` assertions (stub scaffolded + consumes `vigos.lib.mkProjectShell` + preserved on upgrade) hold.

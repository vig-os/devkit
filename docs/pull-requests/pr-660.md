# [PR 660](https://github.com/vig-os/devcontainer/pull/660) ci(security): vulnix + SBOM CVE scanning for the Nix image (#637)

## Summary

Implements **T3.1 (#637)**: a Nix image has no `apt`/`dpkg` DB, so Trivy's OS-package scanner goes dark. This replaces the primary CVE signal with **`vulnix`** (nixpkgs-native), keeps Trivy for an SBOM + defence-in-depth view, re-authors the security policy, and rebuilds the expiry-gated exception register around `vulnix` findings. **This produces the objective go/no-go gate input for #639.**

### What changed
- **Nightly `scan-nix-image` job** (`security-scan.yml`): builds the image package closure (new flake `packages.devcontainerImageEnv`) and runs `vulnix` as the primary scanner; emits a CycloneDX SBOM and a Trivy SBOM-mode vuln view for defence in depth; uploads both scanners' output as `vulnix`-vs-Trivy overlap evidence.
- **`vulnix-gate`** (`packages/vig-utils`, TDD): fails on HIGH/CRITICAL (CVSS v3 ≥ 7.0) `vulnix` findings not covered by a non-expired exception. Reuses `check_expirations.parse_entries`, so **`.vulnixignore` shares the `.trivyignore` `Expiration:` format and one expiry SSoT** (wired into the pre-commit hook + `ci.yml`).
- **Flake**: lifted shared `imageTools` to expose `devcontainerImageEnv` (accurate `vulnix --closure` target — the gzipped OCI tar has no scannable store refs) and a pinned `packages.vulnix`.
- **`docs/CONTAINER_SECURITY.md`**: re-authored for the Nix posture — dropped the `apt --only-upgrade` escape hatch + "why not `apt-get upgrade`" section; "advance the `nixpkgs` rev" is the primary lever; documents dual registers and the residual Debian scan until #642.

### Scope guardrails
- The gate is **non-blocking (`continue-on-error`)** during discovery: `vulnix` over-reports because it can't see `nixpkgs`' backported patches (local run surfaced ~83 HIGH/CRITICAL, mostly false positives). #639 flips it to blocking after triage + nixpkgs advance, and wires SARIF + a dedup issue.
- `.vulnixignore` starts empty (header-only); findings are triaged into it before cutover.

## Verified locally
- `nix build .#devcontainerImageEnv` + `nix run .#vulnix -- --closure ./result --json` → 38 derivations, scores parsed correctly.
- `vulnix-gate` against the real output flags HIGH/CRITICAL sorted by score; **509 `vig-utils` tests pass** (12 new).
- `check-expirations .trivyignore .vulnixignore` → validated.

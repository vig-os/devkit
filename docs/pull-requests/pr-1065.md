# [PR 1065](https://github.com/vig-os/devkit/pull/1065) fix(ci): banner-aware doc identity test and full shape-test coverage in project checks

## Summary

Fixes the compounding pair found while landing #1063: a test that has been silently red on `dev`, and the CI scope gap that let it go unnoticed.

### #1060 — banner-aware identity assertion
`test_scaffold_doc_matches_root_sssot` broke when #1043's banner landed on the scaffold `DOWNSTREAM_RELEASE.md` (confirmed RED on the base commit). Fix: a new public `strip_banner()` helper in `scripts/transforms.py` — the exact inverse of the `Banner` transform, reusing its own header-split and banner-block internals rather than re-encoding the banner shape — and the test compares `strip_banner(scaffold) == root`, still guarding real content drift. Test-only fix for an already-failing test, so a single commit (RED confirmed first).

### #1061 — CI runs the lightweight suites by default
`Project Checks` ran only `tests/test_utils.py` + `packages/vig-utils/tests`; every shape-test file added recently never executed in CI. The `suite: all` step now runs `tests/` with an explicit `--ignore` deny-list of the 9 heavy modules (each needs nix, the built image, or podman, and each is covered by its dedicated targeted CI job — table in the action header). Deny-listing the slow-changing heavy set instead of allow-listing the fast-growing light set means **new shape-test files run in CI automatically** — demonstrated live: after merging dev, the selection swept in #1063's `test_scaffold_lint.py` with no action edit (640 → 672 tests).

Commit order preserves per-commit green: the test fix lands before the scope widens onto it.

## Test evidence
- Exact final CI selection, post-dev-merge: **672 passed in ~10s** (coverage 83%, gate 50%); all #1061-listed files collected; none of the 9 heavy modules collected
- `prek run --all-files` → green (re-verified independently before push)

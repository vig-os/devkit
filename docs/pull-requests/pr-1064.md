## Summary

Completes the #1036 banner coverage — three follow-ups bundled because they share the same mechanism (`Banner` transform, `_BANNER_SKIP`, `PRESERVE_FILES`).

### #1053 — JSONC banners
- `//` comment style added to the Banner transform; `devcontainer.json`, `.vscode/settings.json`, and the workspace example leave `_BANNER_SKIP` and carry the managed banner.
- The strict `check-json` hook gets an exclude for exactly those three paths in `nix/hooks.nix` (SSoT; both renders regenerated, drift-gated).
- Blast-radius handled: the image-gated integration tests parsed `devcontainer.json` with strict `json.load` — now JSONC-tolerant via a conftest helper, keeping Image Tests green.
- The `is_transformed` invariant from #1043 holds automatically: `.vscode/settings.json`'s manifest entry reclassifies as transformed via `_entry_gets_banner`.

### #1054 — `justfile.local` preservation made real
- Added to `PRESERVE_FILES` (its header claimed preservation the mechanism didn't deliver — the #878/#913 silent-clobber class); now carries the **preserved** banner, with the redundant header claim dropped (the banner owns provenance claims). bats coverage for upgrade-preservation.

### #1055 — node seed banner
- New `_SEED_BANNERS` map + `apply_seed_banners()` in sync-manifest stamps seed inputs outside `assets/workspace/`; the variant derives from the `PRESERVE_FILES` target each seed feeds (same SSoT as everything else, tamper-gated, idempotent). A first-scaffolded Node `justfile.project` now opens with the preserved banner (bats).
- gitignore fragments decision (per the issue): **no banner** — they append into a managed `.gitignore` whose base already opens with one; documented in code.
